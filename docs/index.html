Deterministic · bounded · reviewable

Four perspectives. One verifiable run.

QSO-FABRIC coordinates Atlas, Nova, Orion, and Lyra inside a finite research harness. It records ordered events, state freeze hashes, bounded messages, and final proposals without granting the objects shell, network, credential, wallet, or repository authority.

Current status

Blocked — unverified runtime and upstream contracts.

The Python harness exists, but packaging, licensing, cross-environment replay, adversarial fixtures, rollback evidence, security review, provenance, and read-only upstream compatibility remain release gates.

Product boundary

A research integration harness, not an autonomous control plane.

The repository's active task chain prioritizes formal acceptance of the existing runtime. Open-ended learning, production orchestration, payment settlement, credentialed action, and owner-wide repository automation are explicitly outside the current MVP.

Architecture

Explicit inputs, bounded collaboration, inspectable evidence.

A human supplies an objective, deterministic seed, and limits. The fixed four-object loop records every consequential step in a hash-chained ledger and emits one JSON artifact for independent review.

Trust boundary

Proposals remain proposals.

The report records observations, deterministic inferences, messages, integrity metadata, and final proposals. It does not authorize execution or establish that a proposal is correct.

Current prohibited authority

  • Shell or arbitrary command execution
  • Unrestricted network access or external learning
  • Credentials, secrets, wallets, or payment settlement
  • Repository writes or cross-repository mutation
  • Self-modification or production orchestration

The current ledger and freeze hashes are candidate integrity mechanisms. They require versioned contracts, adversarial fixtures, and reproducibility evidence before they should be treated as stable interchange formats.

Developer onboarding

Reproduce the existing baseline before extending it.

Use an isolated Python environment, run the current tests, execute one seeded experiment, and retain the exact commit, interpreter, commands, results, and artifact hashes. These commands are baseline entry points, not evidence that the release gates have passed.

python3 -m venv .venv
. .venv/bin/activate
python -m pip install --upgrade pip pytest
python -m pytest -q
python -m qso_runtime.four_qso_experiment \
  "P0 reproducibility baseline" \
  --seed 2987 \
  --rounds 4 \
  --output artifacts/p0-baseline.json
python -m json.tool artifacts/p0-baseline.json >/dev/null
shasum -a 256 artifacts/p0-baseline.json
