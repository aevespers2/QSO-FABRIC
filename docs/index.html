Four perspectives. One inspectable run.

QSO-FABRIC coordinates Atlas, Nova, Orion, and Lyra inside a finite research harness. It records ordered events, state freeze hashes, bounded messages, and final proposals without granting shell, network, credential, signing, wallet, deployment, repository, format-standard, or canonical-state authority.

Current status

Blocked — unverified runtime, upstream contracts, and gluing ownership.

Packaging, licensing, cross-environment replay, adversarial fixtures, rollback evidence, security review, provenance, read-only upstream compatibility, and format/lifecycle/authority ownership remain release gates.

Current baseline

A research integration harness, not an autonomous control plane.

The active task chain prioritizes verification of the implemented four-QSO runtime. Open-ended learning, production orchestration, payment settlement, credentialed action, portfolio-wide serialization, signing, and owner-wide repository automation are outside the current MVP.

A.L.I.S.T.A.I.R.E. boundary

Bounded experiment evidence flows to human review.

QSO-FABRIC contributes deterministic seeded runs away from the wall-clock timeout boundary, integrity-marked events, proposals, and Nova's contradiction-oriented review posture. Separately chartered owners must retain portfolio planning, repository operations, format and lifecycle standards, capabilities, credentials, merge and deployment policy, audit, signing, and cross-repository rollback.

Cross-repository composition

Candidate contracts do not glue automatically.

PR #16 proposes a disabled QSIO adapter. PR #17 proposes QSO envelopes, composition roots, serialization families, hashes, signatures, and mutation classes. Both intersect with QSO-GENOMES, QuantumStateObjects, qsio-kernel, Repository 1, Bridge, Seeker, and review interfaces. Their local coherence does not decide portfolio ownership.

Material obstruction

PR #17 combines neutral envelope mechanics with genome, runtime, Fabric, transport, mutation, signing, and authority concerns that do not share one natural owner.

Lowest-coupling repair

Use a small neutral envelope and conformance layer, repository-owned payload profiles, and independent Repository 1 capability and canonical-state decisions.

Serialization status

Canonical JSON is the implemented authoring profile. Canonical CBOR, package, stream, and signature profiles remain proposed until byte-level conformance evidence exists.

Required witnesses

Genome → envelope → runtime, envelope → runtime → Fabric, Fabric → Bridge/QSIO → Repository 1, freeze → revocation → recovery, and evidence → interface → correction must commute under fail-closed fixtures.

Developer onboarding

Reproduce the baseline before extending it.

Use an isolated Python environment, run the tests, execute one seeded experiment, and retain the exact commit, interpreter, commands, results, and artifact hashes.

python3 -m venv .venv
. .venv/bin/activate
python -m pip install --upgrade pip pytest
python -m pytest -q
python -m qso_runtime.four_qso_experiment \
  "P0 reproducibility baseline" \
  --seed 2987 --rounds 4 \
  --output artifacts/p0-baseline.json
shasum -a 256 artifacts/p0-baseline.json
